# Architecture: Download (`pyaesa/download/`)

## Purpose

The `pyaesa.download` package owns raw source acquisition for pyaesa owned
datasets. It downloads or generates raw files under the active workspace
`data_raw/` tree and records enough metadata for later processing calls to
validate source coverage and provenance.

This document is for external Python contributors. It describes the public
download surface, canonical source owners, path and metadata boundaries, and the
rules that keep raw acquisition deterministic and testable.

## Public Surface

The package level public API is exported through `pyaesa.__init__`.

| Public function | Owner module | Responsibility |
| --- | --- | --- |
| `download_mrio(...)` | `pyaesa/download/mrios/download_mrio.py` | Download supported raw MRIO sources and record raw year coverage. |
| `download_pop_gdp(...)` | `pyaesa/download/pop_gdp/download_pop_gdp.py` | Download or generate raw population and GDP inputs. |
| `download_ar6(...)` | `pyaesa/download/ar6/download_ar6.py` | Download raw AR6 scenario explorer and historical climate comparison inputs. |

Download functions are user facing and must have complete Google style
docstrings when touched. Internal download helpers must remain under their
family package.

## Responsibility Boundary

`pyaesa.download` owns:

| Area | Canonical owner |
| --- | --- |
| Raw MRIO source registry and year selection | `mrios/utils/source_registry.py`, `mrios/utils/year_selection.py` |
| Raw MRIO path ownership | `mrios/utils/paths.py` |
| EXIOBASE raw acquisition | `mrios/utils/exio_download.py` |
| OECD ICIO raw acquisition | `mrios/utils/oecd_download.py` |
| Raw MRIO metadata and logging | `mrios/utils/metadata.py`, `mrios/utils/logging.py` |
| Population and GDP raw paths | `pop_gdp/raw_paths.py` |
| World Bank, IMF Taiwan, and SSP raw acquisition | `pop_gdp/download_wb.py`, `pop_gdp/download_imf_twn.py`, `pop_gdp/download_ssp.py` |
| Population and GDP raw coverage metadata | `pop_gdp/metadata.py` |
| AR6 raw path and metadata ownership | `ar6/utils/io/paths.py`, `ar6/utils/io/metadata.py` |
| AR6 source acquisition and parsing | `ar6/utils/sources/` |
| AR6 download report payloads | `ar6/utils/io/reports.py` |

`pyaesa.download` does not own:

| Area | Owner |
| --- | --- |
| Creating the active workspace root | `pyaesa/workspace_initialisation/` |
| Processing raw data into runtime assets | `pyaesa/process/` |
| Deterministic aSoCC, aCC, ASR, AR6 CC, or IO-LCA calculations | Respective scientific packages |
| Monte Carlo or Sobol execution | `pyaesa/shared/uncertainty_assessment/` and family uncertainty packages |
| Generic filesystem helpers | `pyaesa/shared/runtime/io/filesystem.py` |

Do not add processing logic to download modules. Download modules may inspect
raw file existence and raw metadata, but processed output decisions belong in
`pyaesa.process`.

Path construction must stay inside the relevant path owner. Do not build raw
paths directly in source handlers or public entry modules when a path helper
exists.

## Raw Data Flow

The download family follows one common pattern:

1. Public function validates public arguments.
2. Family owner resolves the active workspace through path helpers that depend
   on `set_workspace(...)`.
3. Family owner normalizes selectors such as source keys and years.
4. Existing raw files and raw metadata are inspected before work starts.
5. Missing raw assets are downloaded or generated by the source owner.
6. Raw metadata is written with coverage and provenance for later processing.
7. The public function returns a report when work was done or errors were
   captured, and returns `None` when the requested raw scope is already present.

Download functions must not keep module level mutable runtime state across
calls. Any persistent state belongs on disk in documented raw files and raw
metadata.

## MRIO Download Contract

Supported MRIO sources are declared in `mrios/utils/source_registry.py`.

| Source key | Family | Raw scope |
| --- | --- | --- |
| `exiobase_396_ixi` | EXIOBASE | EXIOBASE 3.9.6 industry by industry |
| `exiobase_396_pxp` | EXIOBASE | EXIOBASE 3.9.6 product by product |
| `exiobase_3102_ixi` | EXIOBASE | EXIOBASE 3.10.2 industry by industry |
| `exiobase_3102_pxp` | EXIOBASE | EXIOBASE 3.10.2 product by product |
| `oecd_v2025` | OECD | OECD ICIO v2025 |

MRIO handler contract:

| Handler behavior | Rule |
| --- | --- |
| `download_year(...)` | Owns source family acquisition for one effective unit. |
| `list_existing_years(...)` | Reports raw years visible on disk. |
| `full_exists(...)` | Detects manually supplied complete raw archives or CSVs. |
| Optional `coalesce_pending_years(...)` | Aggregates public years for bundled downloads. |
| Optional `progress_label(...)` | Provides progress labels for coalesced units. |

`download_year(...)` receives the requested public years and a run local
deduplication set from `download_mrio(...)`. A handler must append a year to
`DownloadReportMRIO.downloaded` only after the source owned archive or CSV for
that year exists in the raw scope.

`download_mrio(...)` records downloaded or refreshed years, skipped existing
years, and errors in `DownloadReportMRIO`. Existing complete raw files are
recognized on non refresh runs even when raw metadata is absent, then metadata
is updated to reflect observed raw coverage. `refresh=True` forces the source
handler to reacquire requested units; callers should use it only when they
intend to replace the raw source scope.

MRIO source keys and shared prerequisite roots are owned only by
`mrios/utils/source_registry.py`. Raw source handlers do not own processed MRIO
saved folder names; processed folder names belong to `pyaesa/process/`. A raw
file whose filename matches a source owned pattern but does not contain a valid
year is malformed input and must fail instead of being ignored.

## Population And GDP Download Contract

`download_pop_gdp(...)` coordinates three raw input families:

| Raw family | Owner | Purpose |
| --- | --- | --- |
| World Bank WDI | `pop_gdp/download_wb.py` | Historical population and GDP. |
| IMF Taiwan | `pop_gdp/download_imf_twn.py` | Historical Taiwan population and GDP supplement. |
| SSP | `pop_gdp/download_ssp.py` | Future population and GDP scenarios. |

The public switches `past_years` and `future_years` select which raw families
run. Raw files are written through `pop_gdp/raw_paths.py`, and coverage metadata
is written through `pop_gdp/metadata.py`.

Each selected source owner clears its own CSV and metadata before rebuilding
when `refresh=True`. The public orchestration owner does not accept alternate
download callables; local fixture control for tests belongs at the source read
or source payload boundary.

Processing harmonization and MRIO matching are not part of this package. They
belong in `pyaesa/process/pop_gdp/`.

## AR6 Download Contract

`download_ar6(...)` downloads the raw climate input domain used by
`process_ar6(...)`:

| Raw asset | Owner |
| --- | --- |
| AR6 public scenario explorer CSV | `ar6/utils/sources/download_iiasa.py`, `ar6/utils/sources/explorer_csv.py` |
| Historical PRIMAP and Global Carbon Budget sources | `ar6/utils/sources/historical_sources.py` |
| AR6 historical figure reference | `ar6/utils/sources/ar6_historical_figure_reference.py` |
| Citation text and raw metadata | `ar6/utils/io/metadata.py`, `ar6/utils/io/paths.py` |

The raw download signature is built from `ar6/utils/config.py` defaults:
database, variables, meta columns, and region. Domain filtering by category,
SSP, study period, harmonization, and output variables belongs in
`process_ar6(...)`, not in `download_ar6(...)`.

The IIASA source owner supports the `ar6-public` scenario explorer through the
IIASA run API and the IIASA public archive files endpoint. Both routes produce
the same canonical explorer CSV written by `ar6/utils/sources/explorer_csv.py`;
downstream processing consumes only that local CSV plus raw metadata.
`download_ar6(...)` exposes `manager_url` only to select the IIASA Scenario
Explorer manager endpoint. It does not accept alternate orchestration
callables.

When saved AR6 raw outputs exist, metadata must match the download signature.
Missing or inconsistent metadata is an error because downstream processing
cannot prove provenance.

## Refresh And Existing File Semantics

Refresh behavior is family local:

| Family | Refresh rule |
| --- | --- |
| MRIO | Requested years or coalesced source units are reacquired by the source handler. |
| Population GDP | Selected raw CSVs and coverage metadata are rebuilt by their owners. |
| AR6 | Raw explorer, citation, metadata, and historical source scopes are rebuilt. |

Do not make `refresh=True` delete unrelated raw families or processed outputs.
Do not infer processed completeness from raw download metadata.

## Adding A Raw Source

For a new raw source, contributors must define:

1. The public source key or public function argument.
2. The raw path owner.
3. The source acquisition owner.
4. The metadata payload and provenance fields.
5. The source coverage selector contract.
6. The processing package that consumes the raw files.
7. Tests with local fixtures and no real network calls.

For a new MRIO source, update `mrios/utils/source_registry.py`, source family
download handlers, year selection, process source configs, and downstream
tests. For a new population GDP or AR6 source, update the family raw path,
metadata, processing loader, and user docs in the same change set.

## Testing And Quality Gates

Package tests for download code live under:

| Family | Tests |
| --- | --- |
| MRIO | `tests/package/download/mrio/` |
| Population GDP | `tests/package/download/pop_gdp/` |
| AR6 | `tests/package/download/ar6/` |

Download tests must be deterministic:

1. No real network calls.
2. Use local fixtures, local HTTP servers, or source read boundary fixtures.
3. Cover public report behavior and raw metadata writes.
4. Cover refresh only within the family raw scope.
5. Do not add orchestration callable injection only for tests.
6. Avoid tests that force private states unreachable through public functions.

For touched download owners, run:

1. `python -m ruff check <touched paths>`.
2. `python -m ruff format --check <touched paths>`.
3. `python -m pyright <touched package paths>`.
4. Targeted `pytest` with `--cov=<touched owner> --cov-branch`.

Keep touched owners at 100 percent line and branch coverage.
